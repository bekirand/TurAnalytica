# TurAnalytica 🌴🤖

**Умный агрегатор и анализатор пакетных туров на базе ИИ (OpenAI GPT-4o-mini)**.

TurAnalytica — это современная платформа для подбора и сравнения туристических путевок, которая использует искусственный интеллект для проведения глубокого, независимого и непредвзятого анализа каждого тура. ИИ оценивает туры по 8 ключевым критериям, выделяя скрытые плюсы и минусы, чтобы пользователи могли сделать идеальный выбор.

![Стек технологий](https://skillicons.dev/icons?i=nextjs,react,ts,tailwind,prisma,postgres)

## 🌟 Ключевые возможности

- 🤖 **AI-скоринг туров**: Мгновенный экспертный анализ туров через OpenAI API по уникальной 100-балльной матрице (соотношение цена/качество, климат, безопасность, отель, транспорт и т.д.).
- ⚖️ **Интерфейс сравнения**: Side-by-side сравнение до 3 туров в наглядной таблице с цветовой индикацией победителей по каждому критерию.
- ➕ **Пользовательский CRUD (Добавление своих туров)**: Вы можете вручную или скопировав данные с любого сайта туроператора добавить информацию о своём туре. После добавления можно в один клик запустить AI-анализ этого тура и затем сравнить его с другими турами (в том числе добавленными вами).
- 📌 **Демо-каталог**: Изначально база данных содержит лишь несколько премиальных примеров туров. Они добавлены исключительно для того, чтобы показать, как выглядит и работает проект с уже заполненными данными.
- ❤️ **Личный кабинет**: Сохранение избранных туров, отслеживание личной статистики и удобный доступ к проведенным анализам.
- 🎨 **Современный UI/UX**: Стеклянный интерфейс (Glassmorphism), плавная анимация, микроинтеракции и премиальный темный дизайн.
- 📱 **Адаптивность**: Полностью мобильная верстка.
- 🔐 **Аутентификация**: Безопасный процесс регистрации и входа с использованием NextAuth.js (включая защиту серверных API).

## 🛠 Технологический стек

* **Frontend**: Next.js 14 (App Router), React, TypeScript, Tailwind CSS, Framer Motion, Lucide Icons, Shadcn UI / Radix (кастомный UI kit).
* **Backend**: Next.js Server Actions & API Routes, NextAuth.js v5 (Auth.js) для авторизации и сессий.
* **Database**: PostgreSQL (Supabase), Prisma ORM с пулингом соединений (PrismaPg & PgBouncer).
* **AI Integration**: Текущая реализация использует OpenAI SDK (GPT-4o-mini) через прокси OpenRouter. *Однако архитектура позволяет легко переключиться на любую другую LLM (Claude, Gemini, Llama), в зависимости от возможностей и предпочтений разработчика.*

## 🚀 Как запустить локально

### 1. Клонирование репозитория

```bash
git clone https://github.com/bekirand/tur-analytica.git
cd tur-analytica
```

### 2. Установка зависимостей

```bash
npm install
# или
yarn install
# или
pnpm install
```

### 3. Настройка переменных окружения

Создайте файл `.env` в корне проекта и заполните его по примеру `.env.example`:

```env
# База данных PostgreSQL (Supabase)
# Замените пароль и project-ref на ваши
DATABASE_URL="postgres://postgres.[project-ref]:[password]@aws-0-eu-central-1.pooler.supabase.com:6543/postgres?pgbouncer=true&connection_limit=1"
DIRECT_URL="postgres://postgres.[project-ref]:[password]@aws-0-eu-central-1.pooler.supabase.com:5432/postgres"

# NextAuth Configuration
AUTH_SECRET="ваш-длинный-случайный-секрет" # сгенерируйте через openssl rand -base64 32
AUTH_URL="http://localhost:3000"

# OpenRouter / OpenAI API Key
OPENAI_API_KEY="sk-or-v1-..." 
```

### 4. Настройка базы данных

Выполните миграции Prisma:
```bash
npx prisma db push
```

Сгенерируйте Prisma Client:
```bash
npx prisma generate
```

Заполните базу начальными моковыми (демонстрационными) данными:
Откройте в браузере `http://localhost:3000/api/admin/seed` — это добавит в базу 4 стартовых тура. **Они нужны только для примера**, чтобы показать, как работает приложение. В дальнейшем вы можете добавлять свои реальные туры через интерфейс сайта.

### 5. Запуск сервера разработки

```bash
npm run dev
# или
yarn dev
```

Откройте `http://localhost:3000` в вашем браузере.

## 📁 Структура проекта

- `src/app`: Основная логика роутинга Next.js App Router (каталог, дашборд, детали тура, формы).
- `src/app/actions`: Server Actions для работы с БД (создание туров, лайки, вызовы AI).
- `src/components`: UI компоненты и переиспользуемые блоки (`shared` для бизнес-логики, `ui` для атомарных элементов).
- `src/lib`: Утилиты, инстансы Prisma, конфигурация Auth.js и клиента OpenAI.
- `prisma`: Схема базы данных PostgreSQL.

## 🧠 Система AI-оценок

Проект использует строго заданный промпт для LLM, чтобы избежать "средних" или завышенных оценок. Модель действует как бескомпромиссный эксперт, вынося оценки в формате JSON:
- Не зависит от количества звёзд (3* отель с отличным сервисом может победить плохую 5*).
- Сильно штрафует за неудачный сезон/погоду для отдыха в конкретной стране.
- Анализирует транспорт (чартер vs регулярные рейсы, длительность трансферов).
- Выносит финальный вердикт, плюсы, минусы и детальное текстовое ревью.

## 🚀 Потенциал развития (Roadmap)

Проект имеет огромный потенциал для масштабирования в полноценный travel-startup. Возможные улучшения в будущем:

1. **Автоматический парсинг каталогов**: Интеграция скраперов (Puppeteer/Playwright) или официальных API крупных туроператоров (TezTour, Pegas, Coral) для автоматического наполнения и актуализации базы туров без ручного ввода.
2. **Система бронирования**: Добавление эквайринга (Stripe / YooKassa) для возможности реальной покупки туров прямо на платформе.
3. **Динамическое ценообразование**: Отслеживание изменений цен на туры в реальном времени с графиками падения/роста стоимости.
4. **Интерактивные карты**: Интеграция Google Maps или Mapbox для отображения отелей, ближайших пляжей и инфраструктуры курорта на странице тура.
5. **Пользовательские отзывы**: Разделение оценок на "Оценка ИИ" и "Оценки реальных путешественников" с формированием смешанного рейтинга.
6. **AI-Ассистент (Чат-бот)**: Встраивание диалогового ИИ, который поможет подобрать тур прямо в чате на основе текстовых пожеланий пользователя (Например: *"Хочу в Турцию в августе с двумя детьми, песчаный пляж, до 250 тыс. рублей"*).
